# Telegram Abridger Userbot — Init
Date: 2026-06-02
Status: in progress

## Goal

Userbot (входит в Telegram как пользователь) делает следующее:
- Следит за входящими сообщениями от не-muted аккаунтов
- Считает сообщения от каждого отправителя за скользящее окно COOLDOWN_INTERVAL
- Если счётчик превышает MESSAGE_FREQUENCY_LIMIT — переводит отправителя в muted на 1 час
- Отмечает сообщения прочитанными
- Склеивает буферизованные сообщения через MESSAGE_CONCAT_STRING (по умолчанию ", ")
- Отправляет результат в тот же чат silently (без уведомления) от имени пользователя

## Plan

1. Выбрать библиотеку — Telethon (см. Research)
2. Реализовать одним скриптом `bot.py` с конфигурацией через переменные окружения / `.env`
3. Логика:
   - Event handler на все входящие сообщения
   - In-memory dict: sender_id → deque of timestamps за COOLDOWN_INTERVAL
   - Если len(deque) > MESSAGE_FREQUENCY_LIMIT → mute на 1 час (UpdateNotifySettingsRequest)
   - Буфер сообщений: sender_id → list of texts, flush по таймеру или при mute
   - Mark read: send_read_acknowledge()
   - Send concatenated silently

## Research & Findings

### Выбор библиотеки: Telethon vs Pyrogram

| | Telethon | Pyrogram |
|---|---|---|
| Статус | Активен (v1.43.2, апрель 2026), перенесён на Codeberg | **Заброшен** (нет релизов 12+ мес.) |
| Stars | ~12k | ~4.6k |
| Mute | `account.UpdateNotifySettingsRequest` + `InputPeerNotifySettings(mute_until=...)` | Нет чёткого API |
| Mark read | `client.send_read_acknowledge(message)` | Не задокументировано |
| Silent send | `send_message(..., silent=True)` | `send_message(..., disable_notification=True)` |

**Решение: Telethon.** Pyrogram мёртв; существует форк PyroTGFork, но требует дополнительной проверки. Telethon покрывает все нужные операции напрямую.

### Gotchas (Telethon)
- `dialog.dialog.notify_settings.mute_until` возвращает `datetime`, не Unix float → сравнивать через `.timestamp()`
- Чаты с topics (`dialog.entity.forum == True`) нужно исключать явно: ответ уходит в главную тему
- Зависимости объявлены как PEP 723 inline metadata в шапке `bot.py` → запуск `uv run bot.py`
